1. Use Case 2.1(manage profile )Test Results

Related Use Cases : Test Results
1.       The user accesses My Profile page and selects the Edit Profile option. P
2.       The system presents the user with the tabular form questionnaire where the fields are already filled in with the information provided previously. All the fields are editable. (If no information provided previously, only the user name field is filled in.) P
3.       The user can change the information and submit the changes. P
4.       System updates the changes in the database. p
1a. User is not logged in  
      1.       System prompts the user to log in and redirects the user to the Login page. P
3a. Mandatory Fields are not filled in p
      1.       The system prompts the user to provide all mandatory information. p
      2.       User fills in the required fields and resubmits the form. p
      3.       Steps 1 -2 are repeated until all mandatory information is provided. p
3b. Information provided is incorrect (data type, data format)  
      1.       The system prompts the user to correct the information providing hints. p
      2.       User corrects the information and resubmits the form. p
      3.       Steps 1-2 are repeated until all information provided is correct. p
3c. The user wishes to delete information. p
      1.       All non-mandatory fields can be deleted (left empty). NT
4a. Database can’t be accessed or updated NT
      1.       System generates an error message to inform the user of the problem and asks the user to repeat the action at a later time. NT
